# AI-OCR 機能について（ndlocr-lite 矩形選択半自動化と OCR）

## 1. 機能概要

| 機能                     | 説明                                                                                                                                                                 |
| ------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **半自動矩形選択**       | 「AIで自動矩形選択」ボタンで、現在ページの画像を AI で解析し、**ブロック単位**（見出し・段落・表・図など）の矩形を検出して、読み順で選択状態にする。                 |
| **OCR テキストレイヤー** | ページを表示すると、バックグラウンドでそのページの OCR を実行し、**行単位**で認識したテキストを PDF の上に透明レイヤーとして重ねて表示する。検索・選択しやすくなる。 |

どちらも **[NDL OCR（国立国会図書館の OCR パイプライン）](https://github.com/ndl-lab/ndlocr-lite)** のロジックをもとにして、さらに **ONNX Runtime Web（WASM）** を使って**ブラウザ上で動かしています。**

---

## 2. OCR の全体パイプライン（画像からテキストまで）

```
  [PDFページ画像]
        │
        ▼
  ┌─────────────────────────────────────────────────────────────┐
  │ 1. 前処理（画像の正規化）                                      │
  │    キャンバスから画像データ取得・必要に応じたリサイズ・パディング   │
  └───────────────────────────┬─────────────────────────────────┘
                              │
                              ▼
  ┌─────────────────────────────────────────────────────────────┐
  │ 2. レイアウト検出（どこに何があるか）                           │
  │    「テキストブロック」「行」「表・図・広告」などの領域を検出       │
  │    → このプロジェクトでは DEIM モデルが担当                      │
  └───────────────────────────┬─────────────────────────────────┘
                              │
              ┌───────────────┴───────────────┐
              ▼                               ▼
  ┌─────────────────────────┐     ┌─────────────────────────────┐
  │ 3a. ブロック単位の矩形   │     │ 3b. 行単位の切り出し         │
  │ テキストブロック・表・図  │     │ 検出された「行」を読み順に並べる │
  │ をまとめて矩形化         │     │ （XY-cut による読み順整序）    │
  │ → 半自動矩形選択で使用   │     │ → OCR テキストレイヤーで使用   │
  └─────────────────────────┘     └───────────────┬─────────────┘
                                                  │
                                                  ▼
                                    ┌─────────────────────────────┐
                                    │ 4. 文字認識（行画像 → 文字列） │
                                    │ 各行の画像を 1 本の文字列に変換  │
                                    │ → このプロジェクトでは PARSEQ  │
                                    └───────────────┬─────────────┘
                                                    │
                                                    ▼
                                    [ 行ごとの bbox + テキスト ]
                                    → 画面上の透明レイヤーで表示
```

### パイプラインの要点

| 段階                                     | 役割                                               | このプロジェクトでの担当                     |
| ---------------------------------------- | -------------------------------------------------- | -------------------------------------------- |
| **前処理**                               | 画像をモデルが扱いやすい形にする                   | キャンバスから画像取得・リサイズ・パディング |
| **レイアウト検出（セグメンテーション）** | 「どこにテキスト／表／図があるか」を領域として出す | **DEIM**（ONNX モデル）                      |
| **ブロック矩形・読み順**                 | 検出領域をブロックにまとめ、読み順を付ける         | テキストブロックのグルーピング + **XY-cut**  |
| **文字認識**                             | 行画像 → 文字列                                    | **PARSEQ**（ONNX モデル）                    |

- **半自動矩形選択**: 1 → 2 → 3a まで（ブロック矩形 + 読み順で選択状態にする）。
- **OCR テキストレイヤー**: 1 → 2 → 3b → 4（行単位で認識し、画面上にテキストを重ねる）。

## 3. 技術スタックと役割

| 技術要素                    | 一言での役割                                                                      |
| --------------------------- | --------------------------------------------------------------------------------- |
| **DEIM（ONNX）**            | 画像から「テキストブロック」「行」「表・図」などの領域を検出する。                |
| **PARSEQ（ONNX）**          | 行画像 1 枚を 1 本の文字列に変換する（文字認識）。                                |
| **読み順整序（XY-cut）**    | 検出したブロック・行に上→下・左→右の読み順を付ける。                              |
| **ONNX Runtime Web**        | ブラウザ上で ONNX モデル（WASM）を実行する。                                      |
| **IndexedDB（キャッシュ）** | レイアウト検出結果と OCR 結果をページ単位で保存し、再計算を減らす。               |
| **Web Worker**              | 重い計算をメインスレッドから切り離し、同時 1 タスク・表示中ページ優先で実行する。 |

---

## 4. 全体の構成

| レイヤー                    | 役割（一言）                                                                                     |
| --------------------------- | ------------------------------------------------------------------------------------------------ |
| **UI（メインスレッド）**    | 表示・操作を受け付け、Worker にタスクを渡して結果を反映する。                                    |
| **処理（Web Worker）**      | レイアウト検出・ブロック矩形・文字認識など重い計算を行う。キャッシュがあれば検出をスキップする。 |
| **キャッシュ（IndexedDB）** | レイアウト検出結果と OCR 結果をページ単位で保存し、再表示時の再計算を減らす。                    |

---

## 5. 「単位」の違い（半自動矩形選択 vs OCR テキストレイヤー）

同じレイアウト検出結果を使うが、**扱う単位**が違う。

| 機能                     | 単位     | 一言                                                   |
| ------------------------ | -------- | ------------------------------------------------------ |
| **半自動矩形選択**       | ブロック | 段落・表・図などを 1 ブロック＝1 矩形として選択する。  |
| **OCR テキストレイヤー** | 行       | 行ごとに文字認識し、位置＋テキストをレイヤー表示する。 |

---

## 6. 利用シーン別の流れ

| シーン                   | 一言での流れ                                                                                                                                                 |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **AIで自動矩形選択**     | ボタン押下で、キャッシュがあればそれでブロック矩形を計算、なければレイアウト検出から実行し、読み順で選択表示する。                                           |
| **OCR テキストレイヤー** | ページ表示時にキューで OCR を実行。キャッシュがあればスケールして表示、なければレイアウト検出から行う。レイアウト結果は 1 回だけ計算し、矩形選択と共有する。 |

---

## 7. 設定・モデル・環境

| 項目               | 一言                                                                                                                      |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------- |
| **モデル・設定**   | ONNX モデルは `ndlocr-lite/model/`、設定（クラス名・文字セット）は `ndlocr-lite/config/`。Worker が読み込んで推論に使う。 |
| **WASM スレッド**  | 負荷抑制のため、論理コアの半分・最低 1・最大 4 に制限。                                                                   |
| **マルチスレッド** | SharedArrayBuffer 利用のため、COOP / COEP ヘッダーが必須（開発・本番とも）。                                              |

---

## 8. 座標とキャッシュの考え方

検出はキャンバス解像度の画像で行い、**キャッシュには画像サイズを保存**してある。再表示時は現在の解像度に合わせて座標を伸縮して再利用する。

## 10. まとめ

| 要素                     | 一言                                                                                           |
| ------------------------ | ---------------------------------------------------------------------------------------------- |
| **レイアウトキャッシュ** | DEIM の検出結果を保存。矩形選択と OCR で共有する。                                             |
| **半自動矩形選択**       | キャッシュ or レイアウト検出でブロック矩形を出し、読み順で選択表示する。                       |
| **OCR テキストレイヤー** | キャッシュ or レイアウトから行テキストを出してレイヤー表示。テキスト埋め込みページはスキップ。 |
| **キュー**               | 同時 1 タスクで、表示中ページを優先する。                                                      |
| **WASM**                 | ブラウザ上で DEIM / PARSEQ をマルチスレッド CPU で実行する。                                   |

国立国会図書館 NDL ラボが公開する **NDLOCR-Lite** とその関連成果を参考にして実装しています。

  - リポジトリ: `https://github.com/ndl-lab/ndlocr-lite`

    `https://creativecommons.org/licenses/by/4.0/deed.ja`
- 読み順整序モジュール: 国立国会図書館が公開する **NDLOCR** と同様のモジュールに基づく実装
  - リポジトリ: `https://github.com/ndl-lab/ndlocr_cli`
  - ライセンス: Creative Commons Attribution 4.0 International (CC BY 4.0)  
    `https://creativecommons.org/licenses/by/4.0/deed.ja`  
    （ライセンス全文および依存ライブラリのライセンス一覧は、上記リポジトリ側の `LICENCE` / `LICENCE_DEPENDENCEIES` を参照してください）
